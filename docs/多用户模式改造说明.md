# 多用户模式改造说明

## 改动概述

将 LinkTerm 从预配置 token 的单用户模式，改造为 Agent 自注册的多用户模式。

## 核心变化

### 认证模型

| | 改造前 | 改造后 |
|---|---|---|
| Agent 认证 | 服务端 config.yaml 预配置 token | Agent 自动生成 token，首次连接自动注册 |
| 浏览器认证 | 输入 access_password 获取 JWT | 输入 Agent Token 获取 JWT |
| Token 存储 | 写死在服务端 config.yaml | 动态存储在 agents.json |
| 多 Agent | 需手动在 config 中添加多个 token 条目 | Agent 启动即注册，无需配置 |

### 数据流

```
Agent 首次启动 → 自动生成 token → 保存到本地 config.yaml
                → 连接服务端 → 服务端自动注册到 agents.json

浏览器访问   → 输入 Agent Token → 服务端校验 agents.json → 签发 JWT（含 agent_id）

Token 重新生成 → Agent 生成新 token → 保存本地 → 断开 → 自动重连 → 服务端自动注册新 token
```

## 修改的文件

### proto/message.go

- `AuthPayload` 增加 `Name string` 字段，Agent 认证时上报显示名称

### server/config.go

- 移除 `TokenConfig` 结构体和 `AuthConfig.Tokens` 字段
- `AuthConfig` 新增 `DataDir string`（agents.json 存放目录，默认 `./data`）

### server/auth.go

- 完全重写为 agents.json 动态管理
- 新增 `AgentRecord` 结构体
- `RegisterOrUpdate(token, name)` — Agent 连接时自动注册或更新
- `FindByToken(token, remoteIP)` — 浏览器登录校验（含暴力破解防护）
- `IssueJWT(agentID, agentName)` — JWT claims 包含 `agent_id` 和 `agent_name`
- `ValidateJWT()` — 返回 `(agentID, agentName, ok)` 三个值

### server/handler_agent.go

- `authenticateAgent()` 改为调用 `auth.RegisterOrUpdate()` 实现自动注册
- 移除 sha256Sum 辅助函数（已迁移到 auth.go）

### server/handler_web.go

- `handleAPIAuth` — 请求体从 `{password}` 改为 `{token}`
- `handleAPIAgents` — 根据 JWT 中的 `agent_id` 过滤，只返回该用户有权限的 Agent
- `handleListSessions/handleCreateSession/handleDeleteSession` — 使用 JWT 中的 `agent_id` 限制权限

### server/handler_terminal.go

- `ValidateJWT()` 调用适配新的三返回值签名

### agent/config.go

- Token 为空时自动调用 `GenerateToken()` 生成随机 token（`lt_` 前缀 + 32字节 hex）
- 新增 `Name string` 配置字段，为空时取系统 hostname
- 新增 `SaveConfig()` — 将配置回写 YAML 文件
- 新增 `RegenerateToken()` — 生成新 token 并保存

### agent/tunnel.go

- `authenticate()` 中 `AuthPayload` 增加 `Name` 字段
- 新增 `RegenerateToken()` 方法 — 生成新 token → 断开连接 → 触发自动重连

### server/web/index.html

- 登录表单输入框从 `type="password"` 改为 `type="text"`，placeholder 从"访问密码"改为"Agent Token"

### server/web/js/auth.js

- POST body 从 `{password: xxx}` 改为 `{token: xxx}`

### deploy/config.yaml

- 移除 `auth.tokens` 部分
- 新增 `auth.data_dir` 配置项

## 服务端配置（改造后）

```yaml
listen: ":8080"
region: "default"

auth:
  jwt_secret: "改为随机字符串-至少32位"
  data_dir: "./data"

session:
  max_per_agent: 10
  buffer_size: 65536

heartbeat:
  interval: 15s
  timeout: 45s
```

## Agent 配置（改造后）

```yaml
servers:
  - url: "wss://your-server.com"
    name: "主节点"

# token 为空时自动生成，保存后不需要手动填写
token: ""
name: ""  # 为空时取系统 hostname

auto_connect: true
prevent_sleep: true
max_sessions: 10
```
