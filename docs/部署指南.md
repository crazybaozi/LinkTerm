# LinkTerm 部署指南

## 架构说明

```
手机浏览器  ──HTTPS/WSS──>  服务端（云服务器）  <──WSS──  Mac Agent
```

需要部署两部分：
1. **服务端**：部署在有公网 IP 的云服务器上（Docker）
2. **Agent**：运行在你的 Mac 上

---

## 一、服务端部署

### 1.1 准备

- 一台云服务器（推荐 1C1G 即可）
- 域名（可选，推荐配合 TLS 使用）
- Docker + Docker Compose

### 1.2 部署步骤

```bash
# 克隆或上传项目
cd /opt
git clone <your-repo> linkterm
cd linkterm

# 编辑配置
cp deploy/config.yaml deploy/config.yaml.bak
vi deploy/config.yaml
```

**必须修改的配置项**：

| 配置项 | 说明 |
|--------|------|
| `auth.jwt_secret` | 改为随机字符串（≥32字符） |
| `auth.tokens[0].token` | Agent 连接凭证，Agent 和服务端必须一致 |
| `auth.tokens[0].access_password` | 浏览器登录密码 |
| `auth.tokens[0].name` | 给 Agent 取个名字（如 "MacBook Pro"） |

```bash
# 启动服务
docker-compose up -d

# 查看日志
docker-compose logs -f
```

### 1.3 TLS 配置（推荐）

将证书放在 `deploy/certs/` 目录，修改 `deploy/config.yaml`：

```yaml
tls:
  cert: /etc/linkterm/certs/fullchain.pem
  key: /etc/linkterm/certs/privkey.pem
```

或者使用 Nginx 反向代理（推荐）：

```nginx
server {
    listen 443 ssl;
    server_name term.example.com;

    ssl_certificate /etc/letsencrypt/live/term.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/term.example.com/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_read_timeout 86400s;
        proxy_send_timeout 86400s;
    }
}
```

## 二、Mac Agent 安装

### 2.1 编译

```bash
cd agent
GOOS=darwin GOARCH=arm64 go build -o linkterm-agent .
# Intel Mac 用: GOARCH=amd64
```

### 2.2 配置

创建 `~/.linkterm/config.yaml`：

```yaml
servers:
  - url: "wss://term.example.com"
    name: "主节点"

token: "与服务端 config 中的 token 保持一致"
prevent_sleep: true
```

### 2.3 运行

**手动运行**：
```bash
./linkterm-agent -config ~/.linkterm/config.yaml
```

**开机自启**（推荐）：
```bash
bash deploy/install-agent.sh
```

自启动后日志位于 `~/.linkterm/agent.log`。

### 2.4 防睡眠说明

配置 `prevent_sleep: true` 后，Agent 会通过 `caffeinate` 阻止 Mac 空闲睡眠。

**注意事项**：
- 接电源 + 合盖 → 不会睡眠（推荐用法）
- 纯电池 + 合盖 → macOS 仍可能强制睡眠
- 建议在 Mac 的「系统设置 → 电池 → 选项」中开启「连接到电源适配器时防止自动休眠」

## 三、使用

1. 手机浏览器打开 `https://term.example.com`
2. 输入访问密码 → 连接
3. 进入终端，即可操作 Mac

**推荐添加到主屏幕**：
- iOS: Safari → 分享 → 添加到主屏幕
- Android: Chrome → 菜单 → 安装应用

添加后从主屏幕打开，体验类似原生 App（全屏、无地址栏）。

## 四、常见问题

### 连接后提示 "Mac 离线"
- 检查 Mac Agent 是否在运行
- 检查 Agent 的 token 是否与服务端配置一致
- 检查 Agent 的 server URL 是否正确

### Agent 启动后无法连接服务端
- 检查服务端是否启动（`curl https://your-server.com/health/ping`）
- 检查防火墙是否开放 8080 端口（或 443）
- 如果用 TLS，确认证书有效

### 手机锁屏回来终端卡住
- 这是正常现象，页面恢复后会自动重连
- 如果超过 30 秒没有恢复，请手动刷新页面

### 如何同时操作多个终端
- 点击右上角 ≡ 菜单 → 新建终端
- 在会话列表中切换不同终端
